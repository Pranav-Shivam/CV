Lecture 6 · Population-based local search

Genetic Algorithms, Ant Colony Optimization & NAS

Three nature-inspired methods, one underlying idea: keep many candidate solutions instead of one, and improve them in parallel.

The whole lecture in one image Hill climbing is one stubborn hiker. Beam search is a few hikers who never talk to each other. GA is hikers who fall in love and have kids. ACO is ants leaving smell-trails so the colony slowly finds the best path. NAS is “let’s use GA to design the brain itself.”

Why each method exists

Each row's limitation is the reason the next method was invented.

Module 2 · Genetic Algorithms

The evolution engine

Selection → crossover → mutation. Everything else in GA is bookkeeping around these three steps.

StoryPicture a dating app for chessboards. Each board is a person. Fitter people (more non-attacking queens) get more dates. Couples have kids. Sometimes a kid gets one weird random trait. Repeat until someone's perfect.

Mutation

4-Queens gene values ∈ {1, 2, 3, 4}. If a selected gene is 2, it mutates to 1, 3, or 4.

⚠
Mutation can make a child worse. GA never guarantees improvement every generation — only that good traits tend to survive over many generations.

Most common confusion (general): every state in local search is a candidate answer — the algorithm hopes to improve fitness, but isn't guaranteed to move closer to a perfect solution at every single step.
Module 2 · Ant Colony Optimization

Pheromone-guided search

Ants leave trails. Shorter paths get reinforced faster. The colony "discovers" good routes — without any single ant being smart.

StoryAnts explore randomly at first. A shorter path means a faster round trip — so that ant lays pheromone sooner, and more ants follow it, reinforcing it further. Unused trails fade. But if a longer-path ant gets lucky early, the colony can be misled.

Symbol decoder

SymbolMeaning
τijpheromone on edge i→j
ηij1 / cost(i,j) — shorter edge = bigger η
αimportance of pheromone
βimportance of cost / heuristic
ρevaporation rate
Qconstant for pheromone deposit
fk / Lkcost / length of the tour built by ant k
visitedklist of nodes ant k has already visited
⚠
Don't confuse η (eta) with ρ (rho). η = 1/cost. ρ = evaporation. Different letters, different jobs.

The two core formulas

Pij = ( τijα · ηijβ ) / Σh ( τihα · ηihβ )
numerator = this edge's pull · denominator = sum over all allowed next edges from i
τij(new) = (1 − ρ) · τij(old) + Δτij
Δτij = Q / fk  if ant used edge (i,j), else 0
✓
Every edge evaporates. Only the edge the ant actually walked gets the extra deposit.

Module 3 · Neural Architecture Search

NEAT → DeepNEAT → CoDeepNEAT

Treat a neural network like an individual in a population — let GA evolve the architecture instead of a human hand-designing it.

StoryInstead of trial-and-error by a human, treat each architecture as an individual. Fitness = task performance. Selection, crossover, and mutation evolve better designs automatically.

Bilevel optimization — evolution finds the architecture (the topology/DNA); gradient descent learns the weights inside a fixed architecture. Two different loops, two different jobs.

🧠
NEAT = neuron-level Lego. DeepNEAT = layer-level Lego, messy. CoDeepNEAT = pre-built Lego kits (blueprints) + reusable bricks (modules) — clean and modular.

CoDeepNEAT mechanics

  • Blueprint — macro-architecture graph; each node points to a module species (a placeholder).
  • Module — a small reusable sub-network (e.g. Conv→ReLU→Pool), evolved in its own subpopulation.
  • Phenotype construction — pick a blueprint → fill each node with a chosen module → connect per blueprint → train.
  • Shared fitness — the assembled network's score credits both the blueprint and every module used, so good building blocks spread.

“Stuck at 60% after 50 generations” — model exam answer

  • Richer module pool — add residual / skip-connection and batch-norm modules so deeper nets train without vanishing gradients.
  • Train longer per evaluation — too few epochs makes fitness noisy and unreliable.
  • Increase population size / mutation rate — escape a converged local optimum in architecture-space.
  • Better data — augmentation or more labels; the bottleneck may be data, not architecture.
  • Tune the fitness metric — validation accuracy with regularization, to avoid overfitting masquerading as fitness.

Vague answers (“use a better algorithm”) get zero marks. Always tie each suggestion to a specific mechanism.

Common mistakes to avoid

  • Saying GA always improves every generation — children can be worse
  • Thinking roulette wheel always picks highest fitness — it only raises probability
  • Forgetting 4-Queens max fitness = 6
  • Confusing η (1/cost) with ρ (evaporation rate)
  • Adding Δτ to every edge — only the used edge gets it
  • Claiming ACO guarantees the shortest path — it doesn't
  • Vague NAS improvement answers — always name the specific mechanism
🧠
Final recap: GA = evolution of solutions. ACO = pheromone-guided path building. NAS = GA designing neural nets. All heuristic — good answers expected, perfect global optimum not guaranteed.
